## 檔案結構
```
index.html              ← 首頁（個人簡介 + 專案卡片列表）
project-1.html          ← 鱗屋之中（UX 研究・Prototype 迭代）
project-2.html          ← G-EIGHT 展場（空間認知・展場 UX）
project-3.html          ← 名偵探柯南 IP（服務藍圖・大規模執行）
project-4.html          ← 親職教育手冊（包容性設計・標案策略）
project-5.html          ← 廣告 Dashboard（數據分析・AI 工具應用）
style.css               ← 共用樣式
assets/                 ← 圖片素材資料夾
  covers/               ← 各專案封面圖
  results/              ← 成果截圖
  process/              ← 過程文件（可選）
```

## 視覺風格
- **色調**：溫和暖色調（米白底、赭土橙、奶茶棕、淡磚紅）
- **排版**：多頁式，每頁單欄閱讀，寬鬆白邊
- **字型**：PingFang TC → Noto Sans TC → Microsoft JhengHei（fallback）
- **感覺**：乾淨、有質感、不花俏，讓文字和數據成為主角

## 每個 Case Study 頁面結構
1. **Hero** — 封面圖 + 專案名稱 + 一句話摘要
2. **背景** — 公司/情境說明
3. **問題定義** — 發現了什麼問題
4. **研究與洞察** — 做了什麼調查、發現什麼
5. **設計決策** — 怎麼解決、關鍵決策是什麼
6. **成果** — 數據結果 + 成果圖片
7. **導覽** — 上一個 / 下一個專案

## 素材狀況
- 封面圖：待使用者上傳（每個專案 1 張）
- 成果截圖：待使用者上傳（每個專案 1–2 張）
- 研究過程圖：選配，有再加
- 暫無素材時：用佔位區塊（虛線框 + 說明文字）
