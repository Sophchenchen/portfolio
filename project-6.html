退貨 User Flow 問卷設計

IP 授權活動周邊商品同步開賣，預估若出售量不低，退貨處理將是一個需要提前規劃的問題。在評估委外通路費用結構後，我設計了一套自營退貨申請路徑，並搭配四種情境分路問卷，讓消費者端清楚、執行端有依據。

公司 頂級創意行銷股份有限公司
專案時間 2025
角色 獨立規劃・通路評估・User Flow 設計・問卷設計
背景 IP 授權活動周邊商品販售

背景

  • IP 授權周邊商品與活動同步開賣，並非活動結束後才上架
  • 宣傳期較晚啟動，已不在電影上映熱潮高峰，出售比例可能偏低
  • 即便如此，只要有銷售量，退貨就需提前規劃
  • 公司本身有人力可安排出貨日；備案：出貨量超出預期時聘工讀生

在此條件下，核心決策是：把退貨業務委外給便利商店通路，還是自行建立申請路徑？

問題定義

我整理了委託便利商店通路的費用項目，發現對於出售量尚不確定的 IP 周邊商品來說，委外方案的成本結構存在明顯風險：

📦

棧板費 & 上架費

商品進入通路倉庫須支付固定棧板佔位費，無論最終銷量如何都必須支付，出售量越低、單位成本越高。

↩️

退貨處理費

每筆退貨另計作業費用，通路對退貨商品的倉儲期限也有限制，超期需自行取回或支付額外費用。

📉

彈性不足

委外後難以自行收集退貨原因，缺乏後續選品與庫存預估的改善依據；流程時間節點也完全由通路決定。

核心問題

出售比例可能偏低的情況下，委外通路的固定費用結構會讓成本難以控制。若公司本身已有人力可以處理，自營退貨路徑不只更划算，還能讓我們掌握退貨數據。

設計決策

對比委外通路的費用結構與公司現有條件，最終決策是自行建立退貨申請路徑。

❌ 拒絕：委外通路（便利商店）

  • 棧板費、上架費為固定成本，與銷量無關
  • 每筆退貨另計處理費，彈性低
  • 倉儲期限由通路決定，超期費用額外發生
  • 退貨原因無法自行收集，缺乏後續改善依據

✓ 採用：自營退貨路徑

  • 公司已有可處理出貨與退貨的人力
  • 自行安排出貨日，時間彈性可控
  • 費用結構透明，不產生額外通路費用
  • 透過問卷收集退貨原因，累積後續選品依據

退貨 User Flow 問卷設計

消費者端的接觸路徑：

  • 商品頁面下方商品敘述明列退費規定，消費者下單前即可了解退貨方式
  • 無論哪種退貨情境，一律導向官方粉絲專頁——統一入口，避免分散聯繫造成資訊遺漏

退貨申請路徑 V3 — 從消費者申請到內部審核、商品回收確認的完整決策節點（點擊放大）

問卷分路：依退貨情境導向不同追問

問卷依退貨情境分成四條路徑，不同情境要確認的資訊不同：

📦

未取貨退件

消費者未到指定門市取貨即發生退件。確認取貨地點是否方便、是否有收到通知，以判斷是否需要重新寄送。

🔍

商品損壞確認

收到商品時發現外觀或內容物損壞。確認損壞樣貌，以及是否已透過粉絲專頁聯繫告知，以利後續換貨或退款處理。

📅

七日猶豫期退貨

在鑑賞期內提出退貨申請。收集退貨主因（不符預期、改變心意等），為後續選品與商品描述提供改善依據。

💬

其他原因

不屬於上述三種情境的退貨需求。以開放填答確保邊緣情境也能被記錄，避免資訊流失。

設計邏輯

消費者在問卷中選擇退貨情境的當下，就決定了後續要填哪些問題、客服要走哪條處理路徑。問卷是 User Flow 的延伸——讓每一筆退貨在抵達粉絲專頁之前，就已經帶著足夠的情境資訊。

成果

  • SOP 建立可直接執行的退貨申請路徑，每個決策節點都有明確依據，減少執行人員現場判斷的負擔
  • 成本 完整評估委外通路費用結構後選擇自營路徑，費用可控，人力彈性保留（備案：出貨量大時聘工讀生）
  • 數據 問卷依四種情境分路，系統性收集退貨原因，為後續選品決策與庫存預估提供實際依據
